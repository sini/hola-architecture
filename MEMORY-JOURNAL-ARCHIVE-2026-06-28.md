**Phase 2** (expanded scope = REIMPLEMENT evalModules; workflow weg70i090 done, 29 agents/2.2M tok). Reconciled verdict (HC1-HC7, several refuted in design-improving ways):
- **HC3 REFUTED → host lib.types merge VERBATIM, do NOT reimplement the kernel.** type.merge is mutually recursive with evalModules (submodule.merge = fresh recursive eval), dispatches on runtime value-shape (anything), THROWS (not a total lattice), and is POSITIONAL for same-priority value-merge (listOf [1,2]+[3,4]→[3,4,1,2]). gen-merge demotes from "the kernel" to an OPTIONAL scalar-priority pre-pass (Radul/Datafun/Leijen only cover the scalar lattice). HC4 REFUTED simplifies: call the SINGLE `type.merge loc defs` uniformly (every v2 type's legacy = (self.v2{...}).value); only valueMeta lost (1 consumer apparmor.nix).
- **HC5 REFUTED = critical gotcha:** lib.extend 2-attr injection reaches only ROOT level; extendModules (modules.nix:386) calls the LOCAL let-bound evalModules → bypassed per submodule element. To own cost-center D you must REPLACE the modules.nix body (BE lib.modules.evalModules), not inject.
- **HC2 UPHELD but HC1 REFUTED:** per-host D-hoist (memoize shape-invariant submodule base, re-run per-element value-merge) = 69% byte-identical in a SYNTHETIC (N=250); but delivered net on a REAL demand-forcing toplevel is UNMEASURED (HC1's 67%-hoistable decomposition was a synthetic artifact; real systemd host is demand-merge-dominated, 19 fn/svc forcing one leaf). Only shape-invariant elements share (shorthandOnlyDefinesConfig=true); module-carrying systemd.services elements defeat it → gate+fallback.
- **HC6 UPHELD floors:** real target toplevel.drvPath ≈12.2M copies/4.7M fn-calls; check=false moves copies −0.2%, drvPath byte-identical. All wins NET of this; incrementality intra-eval only.
