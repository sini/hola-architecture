## 1. Context & motivation

E1→E2b proved hola owns `evalModules` **byte-identically vs nixpkgs** (incl. real fleet hosts, microvm). E3c-C1 adds the first **cross-scope eval-result sharing**: evaluate a user-invariant home-manager base once, reuse across users on one host — and, by parameterization, across **hosts** in one fleet eval (C2).

**Perf framing — CORRECTED, honest (the central finding).** cortex's ~35s cold eval is ~94% intrinsic derivation construction; module-eval is ~6% of wall, of which the home-manager `users` sub-eval is a fraction. **Verification established there is NO sound single-host net win to capture:**

| Form | Sound? | Forces (invariant option, N elements) | Measured |
|---|---|---|---|
| Per-element sentinel (v1–v6 §3) | yes (propagating subclass) | **N + 1** (valueBase + N deltaSentinels; no cross-evalModules thunk sharing) | mechanism **4** vs vanilla **3** at N=3 (`/tmp/hola_force_count.nix`) |
| Naive union (clean-under-⋃ ⇒ serve) | **NO** | 2 (N-independent) | unsound: serves wrong value (`/tmp/wf_union_unsound.nix`) |
| Sound universal = ∩ₑ frozenSet(e) | yes | N + 1 (N per-element sentinels) | collapses to the per-element floor |

The per-element `refine` **deep-force re-pays exactly the merge `spliceEval`'s laziness skips** (Nix memoizes nothing across separate `evalModules` fixpoints — `modules.nix:1109` re-wraps `mkOptionDefault` per `evalOptionValue`). So on a single host the mechanism is **net-negative or, at best, a wash** — and the ~1% ceiling means even a hypothetical perfect win is negligible.

**The value is the C2 substrate.** The one place a *sound* force reduction was measured: hoisting a host-invariant base above the per-host `evalModules` in **one `runDenFleet` eval** → **1 force vs N hosts** (`/tmp/wf_c2hoist.nix`: naive 3 hosts = 3 traces; hoisted = 1). There each host eval is ~tens of seconds, so the ∩-sentinel proof cost amortizes against `N × (large shared-eval)` rather than `N × (tiny user-base eval)`. C1 builds and validates the machinery at the cross-element locus where C2's win lives; C1 itself ships **guard-OFF** (C1-D5).

**Shareability — measured** (nix-config @ `8f84aa62`, users `sini`/`shuo`/`will`): G1 disjoint (`will` == base; `shuo` = base+3; `sini` = base+~245). The base reads a per-user value through one path — `apps/shell/zsh.nix:28`, `programs.zsh.dotDir = "${config.xdg.configHome}/zsh"` (← `home.homeDirectory` ← `home.username = mkDefault name`). No structural `loc` reads in the base; ~24 of ~25 aspects invariant. **Most fleet users are empty/near-empty deltas → the empty-delta fast path (§3) is the dominant real-world case.** No nix-config change.

## 3. Mechanism (the spine — the self-healing overlay + cross-element merge)

**Locus shift.** The freeze no longer lives inside the per-element `extendModules` (`vendor/modules.nix:380`). It lives in **two cooperating pieces**:

**(A) The engine `final.types` overlay** (`lib/engine/share-types.nix`, outside `vendor/`). Extends the E1 engine — which today overrides `final.modules` alone — to *also* wrap the submodule combinators, self-healing every reconstruction path so the wrapper survives `fixupOptionType`:

```nix
# lib/engine/default.nix — mkEngine gains the types overlay (shareEnabled-gated):
holaLib = lib.extend (final: prev: {
  modules = import ./vendor/modules.nix { lib = final; };          # E1, unchanged
  types   = prev.types // (if !shareEnabled then {} else
    let
      # self-heal: survive EVERY fixupOptionType reconstruction so the wrapper is not silently dropped.
      heal = orig: orig // {
        _holaShare      = true;                                     # "wrapper SURVIVED" witness — proves survival, NOT that a freeze fired (C1-D8)
        substSubModules = m: heal (orig.substSubModules m);         # the silent-discard trap fix (modules.nix:1470)
        functor         = orig.functor // { type = t: heal (orig.functor.type t); };  # covers typeMerge / binOp
      };
      # the cross-element NAMES-zip join — attr CONTAINERS ONLY (they merge.v2 over element names):
      join = orig: (heal orig) // { merge = orig.merge // { v2 = a: share.crossElementMerge orig a; }; };
    in {
      attrsWith     = args: join (prev.types.attrsWith args);
      attrsOf       = elem: join (prev.types.attrsWith { elemType = elem; });
      lazyAttrsOf   = elem: join (prev.types.lazyAttrsOf elem);
      submoduleWith = a:    heal (prev.types.submoduleWith a);      # SELF-HEAL ONLY; stock merge.v2 (it fires per-element, by-OPTION defs)
    });
});
```

Verified end-to-end on the vendored engine: with `substSubModules` re-wrapped, `_holaShare=true` (i.e. `optionTypeWrapped=true`) *after* a full `attrsOf (submoduleWith …)` eval, the `merge.v2` trace fires seeing all keys (`/tmp/wf_subst_survive.nix`: `V2 loc=[users] keys=[alice,bob,will]`); **without** the re-wrap the wrapper is silently dropped (`/tmp/wf_type_survives.nix` `optionTypeWrapped=ABSENT-after-eval`). The impl MUST self-heal `substSubModules` **and** `functor.type` (the latter covers `typeMerge`/`binOp`; **there is no `type.mergeOptionDecls` method** — `mergeOptionDecls` is a function reconstructing via `typeMerge`/`functor`, already covered — drop it from the re-wrap list). It MUST **NOT** put the names-zip join on `submoduleWith.merge.v2`: post-self-heal that fires **per element** with by-OPTION defs (`/tmp/wf_subwith_heal.nix`), where a names-zip would fabricate bogus "elements" — the join is `attrsWith`/`attrsOf`/`lazyAttrsOf` only; `submoduleWith` is wrapped for survival only with a stock pass-through merge.

**(B) The cross-element merge — `share.crossElementMerge`** (`lib/engine/share.nix`), invoked at `attrsWith.merge.v2` where **all elements' defs are in scope** (`a.defs`, `zipAttrsWith` over names):

```
crossElementMerge orig a =                              # at attrsWith.merge.v2 — ALL element defs in scope
  let
    allNames  = attrNames (lib.zipAttrsWith (n: _: null) (map (d: d.value) a.defs));  # the element keys
    valueBase = share.hoistedBase orig ;                 # the submoduleWith base eval, ONCE — one shared thunk across all keys
    K         = share.keyInvariant valueBase ;           # key-clean ∧ fn-free; once (key-sentinel fused: name+prefix in one eval)
    frozenOf  = name:                                    # SOUND per-element (NEVER clean-under-⋃):
      let paths = share.deltaConfigPaths (share.defsFor name a.defs) ;  # leaf-recursing; imports-expanded
      in if paths == [] then K                           # EMPTY-DELTA ⇒ full-K freeze (the dominant case; no sentinel)
         else share.refine K (share.deltaSentinel valueBase paths) ;    # throw ONLY this element's paths; deepSeq; tryEval-gated
    # THREADING = DEFS-APPEND (verified /tmp/wf_thread_probe.nix: per-key distinct args, valueBase forced ONCE).
    #   NOT substSubModules-per-key — that rebuilds the base N× (/tmp/wf_subst_base.nix), destroying the hoist.
    #   Injected UNCONDITIONALLY (even empty deltas) so the C1-D8 witness holds; the _module.args key is stripped at
    #   vendor/modules.nix:402 before drvPath ⇒ an injected-but-unread arg is inert (C1-A1 proves this).
    synthetic = { file = "<hola-share>"; value = lib.genAttrs allNames (name:
                  { _module.args._holaFrozen = { set = frozenOf name; base = valueBase.config; }; }); };
  in
    orig.merge.v2 (a // { defs = a.defs ++ [ synthetic ]; });  # STOCK merge; each per-element eval reads the arg at :283
```

**C1 (single host, cross-USER):** each element serves its OWN `frozenOf name` (the larger, sound set) from the shared `valueBase`. **NET-NEGATIVE** on a single host (`refine` re-forces — F1) ⇒ guard-OFF (C1-D5); the value is the validated machinery. **C2 (cross-HOST):** the hoistable artifact is `∩ₑ frozenSet(e)` — but the C1 `frozenOf` uses the **C1 sentinel (name+delta, NO specialArgs)**, which **over-admits** for cross-host reuse (`/tmp/wf_specialargs.nix`: `c1_overadmits_for_c2=true`, `hostDerived` differs per host yet is C1-clean). So C2 reuses the same `∩`/defs-append **FORM** but **recomputes `frozenOf` with the WIDENED sentinel** (name+delta+`specialArgs`+cross-host module-deltas, C1-D7) and hoists that invariant subtree above the per-host evalModules (1 force vs N, `/tmp/wf_c2hoist.nix`). The hoist — not the per-element serve — is the win; its net-positivity is an explicit C2 item (§7c).

- **`deltaConfigPaths elem`** enumerates the delta's contribution by **what it writes** (not a name diff): `collectModules` over the element's modules (imports-expanded), then `pushDownProperties` **recursed to precise leaves** (mkIf/mkMerge/mkOverride push down one level then recurse to non-`_type` leaves — the zero-dedupe-cliff fix, §10), union of recursive `config` attr-paths incl. `config._module.args.<n>`. WHNF-forces each *delta* leaf under the real per-element `config`; the whole probe is `tryEval`-wrapped ⇒ any throw ⇒ that element falls back to the reference body. **The computed leaf-path-set is value-derived** (deltaConfigPaths forces leaves) — so the memo key (C1-D7) is `(base-hash, computed probe.value)`, **never** the syntactic written-path-set (`/tmp/wf_value_leak2.nix`).
- **frozenSet substitution** = the owned `declaredConfig` (`vendor/modules.nix:283`, `mapAttrsRecursiveCond` — its `path` arg is presently `_`-discarded but **supplied**) reads the injected `config._module.args._holaFrozen` (when present): `path ∈ set ? getAttrFromPath path base : opt.value`. For a frozen option, by laziness the per-element merge is never forced *in the per-element eval*; the value comes from the hoisted `valueBase` (forced exactly once across keys, `/tmp/wf_thread_probe.nix`). This is a **bounded, documented divergence** in `vendor/modules.nix` (see §2 C1-D2 note / §5) — it breaks the strict `channel-modules-identity` byte-check, which C1-A1 rewrites to a hunk-excluding diff. (On a single host the dedupe saving is offset by the `refine` force — the F1 net-negative; at C2 the host-base force is hoisted above N hosts — the win.)
- **`structureStable`** (delta declares no new top-level option ∧ same `_module.freeformType`) is derived from the `collectModules` walk `crossElementMerge` already runs — **no second `referenceExtend` eval** (the dropped-eval optimization, `/tmp/wf_strat.nix`).
- **Fallback** is the unmodified reference merge (`shareEnabled=false` ⇒ the overlay is `{}` ⇒ E1 exactly) ⇒ the mechanism cannot ship wrong, only inert.

## 4. Soundness — bounded, honest, empirically verified

An option `O` is frozen for element `e` iff `O` forces **deeply** clean under a sentinel throwing **e's own** per-element-varying inputs. **C1 serves each element its own `frozenSet(e)`**; `universal = ∩ₑ frozenSet(e)` (sound for *every* element, the C2 host-shareable FORM — recomputed with the widened sentinel, §3 C2 note).

**The soundness boundary (the central correction).** A value-throw sentinel perturbs config **values only** — never key **presence**, never the **options** attrset, never the value of a **caught** throw. So the sentinel proves invariance **only for reads that OBSERVE the thrown value** (force it and propagate). Two subclasses are clean-under-sentinel yet e-varying — **mis-classified, gate-backstopped, no general pure-Nix detector:**
- **throws-NOT-observed via catch** — `builtins.tryEval` / `import`-fallback over an option read (catches the value-throw). `/tmp/hola_selfcatch.nix`: `perElement_sound_for_bob=false`.
- **throws-NOT-observed via NON-FORCING presence/structure** — `?`/`hasAttr`/`builtins.attrNames`/`intersectAttrs`/`functionArgs` over a **delta-added** key whose container does **not** force its values (the sentinel *introduces* the key's presence by setting its leaf to a throw), or any read of the **options** attrset structure of a delta-touched path. The unsound boundary is **non-forcing containers**: `types.attrs` (raw) and `types.lazyAttrsOf` (`/tmp/hola_typed_vs_lazy.nix`: `raw_attrs.hole=true`, `lazyAttrsOf_str.hole=true`), plus the options attrset and `_module.args` keys. `/tmp/hola_presence.nix`: `O_sound=false` (`?`), `viaNames_sound=false` (`attrNames`).

**SAFE (throws-OBSERVED) ⇒ intrinsically sound:** plain reads, `if`/`mkIf`, comparison, string interpolation, **`or`** (forces the value → hits the throw → propagates), **and presence reads over a strictly-typed `attrsOf <t>`** — the element type-check forces each value, so the throw propagates and it correctly re-merges (`/tmp/hola_typed_vs_lazy.nix`: `typed_attrsOf.frozen=false`). The blanket "bare `?`/`attrNames` are unsafe" over-states it: the discriminator is **whether the read forces a value**, not the syntax. Every claim below carries a probe.

**The boundary is empirical, not closed — and that is the C2 risk.** Two review passes found two unsound channels of the same shape ("the value-throw does not perturb the channel the option reads": catch sidesteps it; non-forcing presence reads structure not value). There may be more non-forcing channels; "throws-OBSERVED" is currently *everything we have not found a counterexample for*. There is a **fundamental cheap-vs-complete tension**: a complete detector would compare an option's value across two real element contexts — which *is* the per-element merge the freeze skips, defeating the win (the differential-check idea is therefore a non-starter). So the bounded claim + gate backstop is the honest consequence of that tension, not a shortcut. **Critically, the parity gate is a *validation-time* tool, not the *production* oracle** (production runs hola only). C1 is guard-OFF ⇒ moot. But **at C2 the freeze is ENABLED, so any undiscovered non-forcing channel is a silent *production* unsoundness with NO backstop.** The real C2-enable question is therefore "what is the *measured frequency* of non-forcing channels in the actual shared base, and is that residual production-acceptable?" — currently unmeasured.

**The closed per-element varying surface.** `submoduleWith.merge.v2` re-enters per element as `base.extendModules { modules = [{_module.args.name = last loc;}] ++ defs; prefix = loc; }` (`types.nix:1449-1455`), `specialArgs` baked into `base` once (`:1393`, delta-independent for C1). So e's varying surface = **{key (`_module.args.name` AND the last `prefix` element, `loc = prefix ++ [name]`, `modules.nix:877`), delta (everything its modules write)}**. The sentinel throws all of it; `deltaConfigPaths` (structural, imports-expanded, leaf-recursing) makes the delta enumeration complete.

**Throws-OBSERVED subclass — SOUND.** For options whose value computation **forces and propagates** throws (plain reads, `if`/`mkIf`, comparison, string interpolation, `or`), forcing clean under e's sentinel ⇒ `O` observes no e-varying input through any channel that forces a value (config-read, module-arg `applyModuleArgs:707-731`, structural `prefix`/`loc`), directly or transitively (a throw in a dependency propagates through the **deep** force). The lemma "clean ⇒ reads no e-varying input through any channel" is **false** for the throws-NOT-observed subclass above (presence/structure/catch sidestep the value). Verified (observing cases):

| Case | Construction | Result |
|---|---|---|
| 1 | normal option, throwing `prefix` | clean (`prefix==[]` short-circuit `:158`) |
| 2 | option reads its `loc` | throws — caught |
| 3 | option reads `name` | throws — caught |
| 4 | `O→D→loc`, no syntactic `.loc` | throws by **deep** propagation, no read-cone |
| 5 | pure option | clean — frozen |
| 6 | real `zsh.dotDir`/`username` (name-derived) vs `editor` | dotDir/username throw+remerge; editor freezes |

**deepSeq is mandatory.** WHNF (`seq`) misses a transitive read buried in a nested attr (`/tmp/hola_e3c_verify.nix` `T5_whnf_misses=true`); only `deepSeq` catches it (`T5_deep_catches=true`). `refine`/`keyInvariant` MUST `deepSeq`-force.

**The naive union is FORBIDDEN — unsound.** `/tmp/wf_union_unsound.nix`: `X = if (tryEval config.p).success then config.q else "safe"`; alice writes `p`, bob writes `q`. Throwing the **union** `{p,q}` makes `tryEval p` fail ⇒ `X="safe"` clean ⇒ union freezes `X` and serves `valueBase` `q-base`; but `realBob_X = q-bob` (`union_sound_for_bob=false`). The per-element bob-sentinel (throws only `q`) correctly throws and remerges. **Throwing more flips throw-catch branches and hides genuine dependencies** ⇒ over-admit in the unsafe direction. Universal MUST be `∩ₑ frozenSet(e)`, never `clean-under-⋃`.

**The throws-NOT-observed subclass — mis-classified, gate-backstopped.** Even the **per-element** sentinel mis-classifies these (it is not just a union problem):
- *catch:* `/tmp/hola_selfcatch.nix` `X = let r = tryEval config.q; in if r.success then "got-${r.value}" else "fallback"`, bob writes `q` ⇒ bob's sentinel throws `q` ⇒ `tryEval` catches ⇒ `X="fallback"` clean ⇒ wrongly frozen (`perElement_sound_for_bob=false`, `got-q-base ≠ got-q-bob`).
- *presence/structure:* `/tmp/hola_presence.nix` `O = if config.settings ? extra then …`, bob adds `settings.extra` ⇒ bob's sentinel sets `settings.extra=throw` which **introduces the key** ⇒ `?` reads presence without forcing ⇒ clean ⇒ wrongly frozen (`O_sound=false`); same for `builtins.attrNames config.settings` (`viaNames_sound=false`).

There is **no general pure-Nix detector** for these inside a thunk. Mitigation: **the required 3-way parity gate (C1-D4) is the backstop** — a wrongly-frozen option that affects a gated host's `drvPath` surfaces as a diff. The class is narrow (explicit `tryEval` / presence-probe of a delta-added key in a config def), but the v1–v6 "intrinsic, hola-only / complete proof" guarantee is **bounded to the throws-OBSERVED subclass** and does not cover it. C1-A2 carries a regression case for each (catch + presence) alongside the forbidden-union guard.

**Function-content rejection.** `deepSeq` does not enter lambda bodies, so a clean value may *contain* a closure capturing a per-element input. `keyInvariant` requires `tryEval(force)` success ∧ a **function-free** forced tree (structural `isAttrs→mapAttrs / isList→map / isFunction→reject`, over the already-forced `tryEval` value — not `toJSON`, which throws uncatchably on functions). This walk doubles as the **serializability admission gate** (C1-D7 / C2): a function-free value `toJSON`-serializes for cross-invocation reuse; a closure-bearing one is intra-process-only.

**No-freeze reassembly baseline (`set={}` everywhere) ⇒ byte-identical** to the reference merge by construction (every `opt.value` normal; the unconditionally-injected `_holaFrozen` arg is unread when `set={}` and is stripped at `:402` before drvPath). This is the C1-A1 invariant — distinct from the **empty-delta ⇒ full-K freeze** path (§3), which freezes *everything* invariant for an element with no delta.

## 7. What it proves / honest unknowns / out of scope

**Proves:** a sound (throws-OBSERVED subclass), self-healing, cross-element freeze mechanism at the type seam, validated byte-identical to nixpkgs; the C1-D8 checks detect a silent no-op; the empty-delta + ∩-form + leaf-recursing pushDownProperties are byte-identical.

**Honest unknowns / corrections:** (a) **no sound single-host net win** (per-element ≥ N+1 forces, union unsound, ceiling ~1%); C1 ships guard-OFF. (b) **throws-NOT-observed subclass** — `tryEval`/`import`-catch over a read AND presence/structure probes (`?`/`attrNames`) of delta-added keys / the options shape — is mis-classified; frequency in real HM/nixpkgs config defs unmeasured; parity-gate-backstopped, not intrinsically sound. (c) **C2 net-positivity — MEASURED net-NEGATIVE under the sound (per-host-sentinel) form** (`/tmp/hola_c2_cost.nix`): vanilla N hosts = **3** base-forces; the sound C2 = 1 hoist + N per-host sentinels = **4** = N+1 > N — the *same* net-negative as single-host. The `/tmp/wf_c2hoist.nix` "1-vs-N" figure measured only the hoist saving and **omitted the N sentinel forces** needed to *prove* host-invariance. C2 is net-positive **only if** the shared base is provable host-invariant with ~**O(1)** sentinels — i.e. it reads `specialArgs` only and **no** host-specific delta path (one `specialArgs`-throw proves all hosts). Whether the *real* fleet base is that isolated is **unmeasured**. ⇒ **A real-fleet C2 force-count is the gating pre-build experiment (§8); do NOT assume net-positive.** (d) **`structureStable` under-approximates** (same root cause as (b)): it checks only *new top-level* option decls, so a delta declaring a *nested* option, or a base option reading the *options structure* of a delta-touched path, slips past a value-throw sentinel — fold into the (b) gate-backstopped class; broadening to any-depth + "no base option observes the options structure of a delta-touched path" is future work. G1 config-only deltas pass.

**Out of scope:** transitive read-cone (refuted — deep throw-propagation suffices for the propagating subclass); **cross-invocation** C2 (CI across days) needs a serialized/CA store (gen-rebuild constructive dim) or IFD absent in pure Nix — only **intra-process** fleet sharing (one `runDenFleet` eval, same heap) is pure-Nix-feasible; no nix-config change.

## 10. Risks & mitigations

| Risk | Mitigation |
|---|---|
| **Silently-discarded overlay** (`fixupOptionType`→`substSubModules`/`typeMerge`/`functor.type` drop a naive wrap ⇒ green build, 0% win). | Self-heal **every** reconstruction path; the C1-D8 three-check (`_holaShare` survival + `frozenOf`-nonempty proxy + `ci#share-fired` trace) — verified necessary (`/tmp/wf_type_survives.nix` vs `/tmp/wf_subst_survive.nix`). |
| **Naive union (unsound)**. | **Forbidden by construction** — universal is `∩ₑ frozenSet(e)`, each throwing only that element's paths; regression-test rejects `clean-under-⋃` (`/tmp/wf_union_unsound.nix`). |
| **Throws-NOT-observed option wrongly frozen** — `tryEval`/`import`-catch over a read (per-element too, `/tmp/hola_selfcatch.nix`) AND presence/structure probes `?`/`attrNames` of a delta-added key or the options shape (`/tmp/hola_presence.nix`). | No pure-Nix detector ⇒ **bounded soundness claim** (throws-OBSERVED subclass) + the required 3-way parity gate as the backstop (surfaces as a per-user drvPath diff); guard-OFF on single host; `or` is safe. |
| **Silent no-op masquerades as success** (overlay survives but freezes nothing). | C1-D8 three-check: `_holaShare` (pure) + `frozenOf`-nonempty proxy (pure) + `ci#share-fired` trace (impure); a pure flake-check alone cannot assert an invocation count. |
| **Under-throwing the delta** (missed contribution ⇒ wrongly frozen). | `deltaConfigPaths` enumerates **written** config-paths (catches additions, `imports`/`mkMerge`, base-arg value-overrides), leaf-recursing; property-tested; parity-gate backstop. |
| **pushDownProperties zero-dedupe cliff** for structured parents (throwing a parent collapses the whole element's frozenSet — `/tmp/wf_pushdown.nix` `coarse_frozen=[]`). | Recurse `pushDownProperties` to precise leaves (`/tmp/wf_pushdown2.nix` `fine_invariant=true`); sound over-approx retained. |
| **specialArgs assumed invariant** (C2-blocking). | Invariant for C1; the parameterized throw-surface (C1-D7) **widens** to throw `specialArgs` at C2 (`/tmp/wf_specialargs.nix`). |
| **Memoization keyed by syntactic path-set** (unsound — value-derived). | Key on `(base-hash, computed probe.value)` (C1-D7), not the path-set (`/tmp/wf_value_leak2.nix`). |
| **Win negligible / zero on single host.** | Acknowledged (C1-D5 guard-OFF); the validated mechanism + C2 substrate stand. |
